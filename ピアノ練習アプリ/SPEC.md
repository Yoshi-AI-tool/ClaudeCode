# ピアノコード練習アプリ 仕様書

## 基本方針

- 単一HTMLファイルで完結（ビルド不要、外部依存なし）
- 動作環境：Google Chrome のみ（Web MIDI APIの制約）
- タブで「フラッシュカード」「コード進行」を切り替え
- カラー：正解=緑 `#1D9E75`、不正解=赤 `#E24B4A`、アクセント=青 `#378ADD`

## Phase 2（完成）

### フラッシュカードモード

コード名が表示され、MIDIキーボードまたはマウスクリックで答える練習モード。

#### 設定項目

| 項目 | 内容 |
|------|------|
| 出題方式 | 完全ランダム / キー指定 |
| キー（キー指定時） | C〜B、長調/短調 |
| 出題範囲（キー指定時） | ダイアトニック / 自由 |
| 出題コード | チェックボックス式（複数選択） |
| 5th省略 | 不可（デフォルト）/ 可 |
| 判定時間 | 0.1〜1.0秒（0.1刻み） |

#### 出題コード一覧

**三和音**：メジャー、マイナー、dim、aug、sus4  
**7th系**：7、M7、m7、mM7、dim7、m7♭5

#### 出題方式の詳細

**完全ランダム**：全12ルート × チェックしたコード種を全組み合わせで出題

**キー指定 - ダイアトニック**：そのキーで自然に発生するコードのみ出題
- Cメジャー例：C, Dm, Em, F, G, Am, Bdim（＋7th版）

**キー指定 - 自由**：スケール音をルートに、チェックしたコード種を全組み合わせで出題
- C7など本来ダイアトニックでないコードも練習できる

> **経緯**：もともと難易度（初級/中級/上級）で出題コードを固定していたが、  
> 「自分で選びたい」という要望でチェックボックス式に変更。  
> キー指定モードで7thコードが出なかった問題→ダイアトニックと7th配列をマージして対応。  
> C7がCメジャーキーのダイアトニックに含まれない問題→「自由」モードで対応（音楽的にもセカンダリードミナントとして使われる）。

#### コード判定ロジック

```
押鍵MIDIノート → % 12 → ピッチクラスセット
↓
必須音が全部揃っているか？ → NO → タイマーリセット（判定なし）
↓ YES
余分な音があるか？    → YES → 赤タイマースタート（不正解待ち）
↓ NO
緑タイマースタート（正解待ち）
↓ 判定時間経過
正解 or 不正解を確定
```

- オクターブをまたいだ転回形は常に正解（ピッチクラス比較のため）
- ペダル保持音も判定に含む（`physPressed` + `activeNodes[m].sustaining`）

#### 5th省略の実装

```javascript
// CHORD_TYPES の fifth フィールドで5thの半音数を定義
// 省略不可（デフォルト）: optionalにある場合もeffRequiredに追加
// 省略可: requiredから5thを除外
```

- dim, aug, sus4, dim7, m7♭5 は fifth: null（省略設定が影響しない）

#### コード定義

```javascript
const CHORD_TYPES = {
  'maj':  { label: '',      required: [0,4,7],    optional: [],   fifth: 7 },
  'm':    { label: 'm',     required: [0,3,7],    optional: [],   fifth: 7 },
  '7':    { label: '7',     required: [0,4,10],   optional: [7],  fifth: 7 },
  'M7':   { label: 'M7',   required: [0,4,11],   optional: [7],  fifth: 7 },
  'm7':   { label: 'm7',   required: [0,3,10],   optional: [7],  fifth: 7 },
  'dim':  { label: 'dim',  required: [0,3,6],    optional: [],   fifth: null },
  'aug':  { label: 'aug',  required: [0,4,8],    optional: [],   fifth: null },
  'sus4': { label: 'sus4', required: [0,5,7],    optional: [],   fifth: null },
  'mM7':  { label: 'mM7',  required: [0,3,11],   optional: [7],  fifth: 7 },
  'dim7': { label: 'dim7', required: [0,3,6,9],  optional: [],   fifth: null },
  'm7b5': { label: 'm7♭5', required: [0,3,6,10], optional: [],   fifth: null },
};
```

#### スコア

正解 / 不正解 / 正解率 / 連続正解 / パス（パスボタンで次の問題へ、スコアには含まれない）

#### ヒント機能

- 💡 ヒントボタンで鍵盤をハイライト
- ルート音：オレンジ（`#FFD0A0`）
- その他の構成音：黄色（`#FFF3CD`）
- 5th省略設定に連動（省略可なら5thはハイライトしない）
- ヒントON/OFFをトグル

#### 出題の均一化

Fisher-Yates シャッフル + デッキインデックスで偏りなく出題（純粋なランダムを廃止）

#### 音名表示

シャープ（C#）とフラット（D♭）を50%の確率で交互に表示（`rootName()`関数）

#### ピアノ鍵盤UI

- 白鍵：28px幅、108px高さ
- 黒鍵：17px幅、68px高さ
- 全鍵盤に2px黒縁取り（`box-shadow: 0 0 0 2px #000`）
- Cの音名ラベル付き

## Phase 3（完成）

### コード進行モード

コード進行をループ再生し、各コードを正しく弾く練習モード。

#### プリセット進行（10種）

1. カノン (Cメジャー)
2. J-POP王道 (Cメジャー) - C-G-Am-F
3. 4-5-6-1 (Cメジャー) - F-G-Am-C
4. ポップ進行 (Cメジャー) - F-G-Em-Am
5. 小室進行 (Aマイナー) - Am-F-G-C
6. vi-IV-I-V (Cメジャー) - Am-F-C-G
7. ロック (Aマイナー) - Am-G-F-E
8. ツーファイブワン (Cメジャー) - Dm7-G7-CM7
9. 3-6-2-5 (Cメジャー) - Em7-Am7-Dm7-G7
10. 12小節ブルース (C)

#### グリッドエディタ

- 小節×拍のテーブル表示
- セルをクリック → ルート音＋コード種ドロップダウン → 確定
- コードが変わるまで前のコードが継続（「・」で表示）
- 「クリア」でセルをリセット（スロット0はクリア不可）
- ＋拍/－拍で拍子変更、＋小節/－小節で小節数変更
- 練習中はグリッド編集不可

#### カスタム進行の保存

- 名前をつけてlocalStorageに保存・読込・削除
- セル編集後に「💾 保存」ボタンが出現

#### テンポモード

**メトロノーム**：BPMに従い自動進行
- ビートバーがBPMに合わせて満タンになると次の拍へ
- ループ終了後にBPM自動調整：全正解→BPM上昇、ミスあり→BPM低下

**自由**：正解コードを判定時間キープすると次の拍へ進む
- フラッシュカードの判定時間設定を共用

#### 自動BPM調整（メトロノームモード時）

- 上昇幅・下降幅・最小BPM・最大BPMをユーザー設定
- デフォルト：上昇+5、下降-10、最小40、最大200

#### 練習中の表示

- 現在のコード名と構成音
- グリッドのアクティブ拍ハイライト（緑）
- スコア：ループ数、正解、不正解、現在BPM

#### 結果画面（終了後）

- ループ数、正解率、最高到達BPM
- 苦手なコード一覧（ミス回数付き）
- 「最初からループ」ボタン
- 「弱点練習 →フラッシュカード」ボタン（ミスしたコードをフラッシュカードモードへ送る）

## Phase 4（未実装・計画中）

### 運指最適化

- 表示タイミング：正解判定後（振り返り式）
- 前のコードからの指の移動距離が最小になる転回形を探索
- 鍵盤上に指番号（1〜5）＋推奨転回形テキストを表示
- 片手 / 両手を選択可能（両手時は左手ルート・右手コード）

### バッキング練習

- お手本パターンを鍵盤アニメーション＋音で表示
- 正誤判定なし（見て真似るだけ）
- パターン：ブロックコード、アルペジオ上行/下行、Albertiバス、ポップス、ジャズコンピング
- コード進行モードと連動、BPMも共有
- 左手・右手パターンを個別に設定可能
