# ピアノ練習アプリ Phase 5 仕様書

## 概要

**テーマ：右手トライアド方式によるヒント表示**

左手がルートを弾くことを前提に、右手のボイシングを「トライアド（3和音）の形」で覚えさせる。
理論（構成音）と演奏（形）を結びつけるショートカット的な覚え方を、アプリのヒント表示に落とし込む。

---

## 背景・設計思想

### 「形で覚える」アプローチ

7thコードを理論から理解するより、右手のトライアド形として身体に入れる方が実践的。

| コード | 構成音（Cを例） | 右手トライアド |
|---|---|---|
| Cmaj7 | C-E-G-B | Em（E-G-B） |
| C7 | C-E-G-B♭ | Edim（E-G-B♭） |
| Cm7 | C-E♭-G-B♭ | E♭（E♭-G-B♭） |
| Cm7♭5 | C-E♭-G♭-B♭ | E♭m（E♭-G♭-B♭） |

→ 法則：**右手 = ♭IIIまたはIIIから始まるトライアド**

### 優先する4種類（Phase 5 の主対象）

実用上この4種で大半のポップス・ジャズをカバーできる。

1. **maj7**（Maj7）
2. **7**（ドミナント7th）
3. **m7**
4. **m7♭5**

---

## 機能仕様

### 1. ヒントのテキスト表示（フラッシュカードモード）

現在のヒント（鍵盤ハイライトのみ）に加え、コード名の下に以下を表示する。

```
Fmaj7
ヒント：左 F ＋ 右 Am
```

- 左手：ルート音名
- 右手：トライアド名（例：Am、Adim、A♭、A♭m）
- 対象コード：maj7 / 7 / m7 / m7♭5 の4種（他のコードはトライアド表示なし）

#### トライアド導出ルール

| コード種 | 右手トライアド | 構成 |
|---|---|---|
| maj7 | IIIm | 長3度上のマイナートライアド |
| 7 | IIIdim | 長3度上のディミニッシュトライアド |
| m7 | ♭III | 短3度上のメジャートライアド |
| m7♭5 | ♭IIIm | 短3度上のマイナートライアド |

#### 表示例（各ルートで）

| 問題 | ヒント表示 |
|---|---|
| Abmaj7 | 左 A♭ ＋ 右 Cm |
| E7 | 左 E ＋ 右 G♯dim |
| Dm7 | 左 D ＋ 右 F |
| Bm7♭5 | 左 B ＋ 右 Dm |

---

### 2. 鍵盤ハイライトの改善（フラッシュカードモード）

現在：構成音を単色でハイライト  
Phase 5：右手トライアドの3音を**まとまりとして色分け**

| 表示 | 色 | 意味 |
|---|---|---|
| 左手ルート | オレンジ `#FFD0A0` | 左手で弾く音 |
| 右手トライアド | 黄色 `#FFF3CD` | 右手で弾く3音のまとまり |

※ コード進行モードの既存ヒントとは別実装（フラッシュカード専用）

---

### 3. ビギナーモード（フラッシュカード設定）

現在：11種コード × 12ルートから出題  
Phase 5 追加：「4種モード」を設定に追加

- **4種モード ON** にすると出題コードを maj7 / 7 / m7 / m7♭5 の4種に絞る
- 既存のコード種チェックボックスとは独立したトグルスイッチ

---

## UI 追加要素

### ヒントテキスト

- 配置：コード名（大文字）の直下
- フォントサイズ：コード名より小さく（例：14px）
- 色：グレー系（目立ちすぎない）
- ヒントOFF時は非表示

### 4種モードトグル

- 配置：フラッシュカードモードの設定パネル内
- ラベル：「7th 4種モード」
- ON にすると他のコード種チェックボックスを無効化（グレーアウト）

## 実装メモ

### トライアド名を生成する関数（新規）

```js
function getTriadHint(root, chordType) {
  // root: 0〜11のピッチクラス
  // chordType: 'M7', '7', 'm7', 'm7b5'
  // 戻り値: { left: "A♭", right: "Cm" } or null（対象外コード）
  const triadMap = {
    'M7':   { interval: 4, triad: 'm'   },
    '7':    { interval: 4, triad: 'dim' },
    'm7':   { interval: 3, triad: 'maj' },
    'm7b5': { interval: 3, triad: 'm'   },
  };
  // ...
}
```

### 既存コードとの関係

- Phase 4 の `applyProgHint`（コード進行モード）はそのまま維持
- Phase 5 のヒントは **フラッシュカードモード専用** として別関数で実装
- 判定ロジック（`checkChord`）は変更なし

## 未対応・スコープ外

- maj / m / dim / aug / sus4 / mM7 / dim7 へのトライアドヒント（Phase 5 では対象外）
- コード進行モードへのトライアドテキスト表示（Phase 5 では対象外）
- 12キー全て均等出題の強制（既存の Fisher-Yates シャッフルを維持）
